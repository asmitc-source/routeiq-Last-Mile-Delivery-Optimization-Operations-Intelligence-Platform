## 📌 Project Overview

RouteIQ is a **production-grade operations analytics platform** built for last-mile delivery networks. It simulates and analyses the kind of data generated by quick-commerce platforms (think Swiggy Instamart, Zepto, Blinkit) operating across 8 Indian metros.

The platform provides:
- **Real-time KPI monitoring** across 12+ operational metrics
- **Predictive ML models** for late delivery risk and SLA breach probability
- **Geospatial analytics** for delivery heatmaps and zone clustering
- **Automated recommendations** generated by an ops intelligence engine
- **Interactive Streamlit dashboard** with city-level drill-downs

## 📊 Features

### Operational Analytics
| Feature | Description |
|---------|-------------|
| Route Inefficiency Analysis | Measures deviation from optimal route (score vs 1.0 baseline) |
| SLA Breach Detection | Real-time monitoring with CRITICAL/WARNING/OK thresholds |
| Rider Utilization | Fleet efficiency scoring with redistribution alerts |
| Dark Store Load | Capacity utilization with load balancing recommendations |
| Peak Hour Stress | Demand surge scoring with pre-positioning recommendations |
| Weather Impact | Monsoon/fog disruption scoring with contingency triggers |
| Delivery Clusters | K-means zone clustering for delivery polygon optimization |
| Geo Heatmaps | Hexbin aggregation of delivery density & late-rate by zone |

### KPIs Tracked (12+)
- ✅ On-time delivery rate
- ⏱ Average delivery time (minutes)
- 🔴 SLA breach percentage
- 📉 Average delay (late orders)
- 🗺 Route inefficiency score
- 🏍 Rider utilization %
- 📦 Daily delivery throughput
- ⚡ Peak-hour stress index
- 💰 Cost per shipment (₹)
- 🚦 Congestion impact factor
- 🌧 Weather disruption score
- 🏭 Dark store utilization %
- ⏳ Fulfillment latency (min)

## 🚀 Quick Start

### 1. Clone & Install
```bash
git clone https://github.com/yourusername/routeiq.git
cd routeiq
pip install -r requirements.txt
```

### 2. Run Pipeline (generates data + trains models)
```bash
# Sample mode: 50k records, ~30s
python run_pipeline.py

# Full mode: 1M records, ~10 min
python run_pipeline.py --full
```

### 3. Launch Dashboard
```bash
streamlit run dashboard/app.py
```

Dashboard available at `http://localhost:8501`

## 💡 Recommendations Engine

The platform auto-generates operational recommendations including:

- 🔴 **SLA breach alerts** with immediate mitigation actions
- 🏍 **Rider redistribution** when utilization is outside 55–90% band
- 🏭 **Dark store load balancing** with store-to-store redirect logic
- 🗺 **Route optimization** via dynamic clustering vs static zones
- 🚦 **Congestion-aware dispatching** with early dispatch triggers
- ⚡ **Peak hour pre-positioning** using predictive demand heatmaps
- 🌧 **Weather contingency protocol** with automated SLA extensions
